# WLED 光效建议

Agent Light 服务端只调用 WLED preset ID，不下发具体效果参数。你需要在每台 WLED 设备上提前保存 1-4 号 preset。

## Preset 约定

| state | server payload | WLED preset ID | WLED preset 名称 |
| --- | --- | --- | --- |
| `idle` | `T=1&PL=1` | 1 | `Agent Idle` |
| `thinking` | `T=1&PL=2` | 2 | `Agent Thinking` |
| `busy` | `T=1&PL=3` | 3 | `Agent Busy` |
| `approval` | `T=1&PL=4` | 4 | `Agent Approval` |

WLED preset 名称只是给你在 UI 里识别用，MQTT 实际调用的是 preset ID。

## 保存方式

```text
1. 在 WLED 首页调好颜色、效果、速度、亮度
2. 打开 Presets
3. 新建或覆盖指定 ID
4. Name 填 Agent Idle / Agent Thinking / Agent Busy / Agent Approval
5. 勾选保存当前状态
6. Save
```

## AI 风格方向

目标不是普通红黄绿状态灯，而是“AI 正在流动”的桌面装置感：低亮、柔和、带拖尾、带层次，在需要用户注意时才明显提醒。

| 状态 | 动画氛围 | 推荐颜色 |
| --- | --- | --- |
| `idle` | 青绿/蓝紫极光缓慢漂移，像在线但安静的核心 | 绿、青、暗蓝 |
| `thinking` | 暖金核心 + 青蓝微粒慢速漂移，有呼吸和星尘 | 金、青、淡紫 |
| `busy` | 蓝紫高速数据流，带拖尾、扫描和轻微闪点 | 蓝、紫、柔白 |
| `approval` | 红橙能量门快速脉冲，中间夹白色闪点 | 红、橙、白 |

## 不同灯型建议

| 灯型 | 像素数 | 推荐表现 |
| --- | --- | --- |
| 单个灯 | 1 | 靠颜色、亮度曲线和呼吸速度表现状态 |
| 2x2 方形 | 4 | 四象限脉冲、对角线呼吸、整体爆闪 |
| 4x4 方形 | 16 | 中心扩散、边框能量场、低分辨率数据流 |
| 8x8 方形 | 64 | 主推荐显示形态，适合极光、粒子、波纹、数据雨 |
| 12 环形 | 12 | 旋转能量环、粒子追逐、完成扫圈 |
| 6 位条形 | 6 | 数据流、进度扫描、左右波、流星拖尾 |

## 8x8 方阵建议

8x8 WS2812 有 64 个像素，是最适合作为主状态灯的形态。它比 4x4 更能表现流动和层次，又比 16x16 更省电、更容易供电和散热。

建议加 2mm-3mm 乳白亚克力扩散板，灯珠到扩散面保持 8mm-15mm。

| 状态 | 8x8 表现 |
| --- | --- |
| `idle` | 深蓝底色上叠加青绿/紫色低速噪声云，中心缓慢呼吸 |
| `thinking` | 中心暖金核心缓慢呼吸，周围青蓝粒子随机漂移 |
| `busy` | 蓝紫扫描线横向流动，局部白色高亮像数据包，旧像素拖尾 |
| `approval` | 外圈红橙边框快速脉冲，中心 2x2 白点闪烁 |

## 12 环形建议

当前默认硬件是 12 灯珠 WS2812B 环形灯珠：

| 状态 | 12 环形表现 |
| --- | --- |
| `idle` | 青绿/蓝紫低亮慢速绕环 |
| `thinking` | 单点慢旋转，金色为主，带柔和拖尾 |
| `busy` | 三点追逐，蓝紫拖尾 |
| `approval` | 红橙双向脉冲，亮度更高但不要长期满亮 |

## 亮度建议

常态亮度建议低一些，避免桌面太刺眼：

```text
idle/thinking: 40-90
busy:          70-120
approval:      120-180，短时间提醒
```

具体亮度按扩散罩、灯珠密度和环境光调整。
